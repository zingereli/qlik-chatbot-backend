# שיפורים עתידיים — Qlik Chatbot

נקודות לשיפור שזוהו במהלך הפיתוח, לפי תחום.

## עלות וביצועים
- **Context Caching ל-system prompt** — ~95% מהטוקנים הם קלט חוזר (ה"ספר הוראות" נשלח בכל בקשה, ~9K טוקנים). זה החיסכון הפוטנציאלי הגדול ביותר.
- מודל קטן/זול ללוקאפ, ושמירת מודל חזק רק למצב ניתוח.
- קיצור ה-prompt / צמצום דוגמאות few-shot.
- Cache לתוצאות שאלות נפוצות (כדי לחסוך קריאות חוזרות).

## יכולות מוצר
- **מדדי ספירה (Family B)** — משרות טק, סה"כ מועסקים, מועסקים בטק, דורשי עבודה. דורש בירור semantics של התקופה/latest (מבנה שונה ממדדי השיעור).
- **חיזוי סדרות-זמן** — תחזית שיעור תעסוקה/שכר (כרגע רק מגמה היסטורית).
- שיפור מצב הניתוח — יותר סוגי חתכים, ציון רמת ביטחון, טיפול טוב יותר בהשוואות מורכבות.
- זיכרון שיחה — תמיכה בשאלות המשך והקשר.
- הרחבה ליותר אפליקציות (הוספת prompts ל-registry).

## אבטחה והרשאות
- **מסלול B** — להחליף את הטוקן-בקוד ב-Load Balancer + Cloud Armor / API Gateway; **רוטציה ל-BACKEND_TOKEN**.
- Rate-limit פר-משתמש (כיום רק פר-IP).
- **Section Access** ב-Qlik — אבטחת שורה כך שכל משתמש יראה רק את הנתונים שמותרים לו (הצ'אט יכבד אוטומטית).

## ניטור ותפעול
- **Budget alert** ב-Billing (התראת עלות ב-$ בנוסף להתראת הטוקנים).
- סף התראה יומי בנוסף לשעתי הקיים.
- הצגת ספירת טוקנים/עלות קטנה בתוך ה-UI של הצ'אט.
- CI/CD לפריסה אוטומטית (כיום פריסה ידנית דרך gcloud).
