# סיכום פרויקט: Qlik Chatbot (POC עובד)

> מסמך זה מסכם מה נבנה, מה עובד, העלויות, ואיך להעביר לפרודקשן ב-GCP.
> תאריך: 31/05/2026

---

## 1. מה בנינו - הארכיטקטורה שעובדת

```
[Qlik Sheet]
  └─ Extension (chatbot.js)  ← רץ בדפדפן של המשתמש
        │ 1. שולח שאלה בעברית
        ▼
[Backend Node.js / Express]  ← רץ בענן (כרגע Render)
        │ 2. קורא ל-Claude API עם ה-schema של האפליקציה
        ▼
[Claude API]  → מחזיר JSON: { measure, dimensions }
        │ 3. חוזר ל-Extension
        ▼
[Extension]  → מריץ query על Qlik (Capability API: createCube)
        │      → שולף נתונים מתוך Qlik (in-memory)
        ▼
[טבלה מוצגת בצ'אט]
```

### 4 הדברים הקריטיים שגרמו לזה לעבוד
1. **Backend בענן (לא localhost)** — ה-CSP של Qlik חוסם חיבורים לכתובות מקומיות/לא מאושרות.
2. **CSP whitelist ב-Qlik** — הוספת הדומיין של ה-backend ב-Management Console → Content Security Policy → directive `connect-src`.
3. **Schema אמיתי** — שליפת ה-measures והשדות האמיתיים מהאפליקציה (במקום להמציא שמות שדות → שגרם להחזרת 0).
4. **Qlik Capability API** — שימוש ב-`app.createCube()` להרצת ה-query (ולא `createSessionObject` שלא קיים ב-Capability API).

## 4. עלויות — פירוט מלא

### 4.1 Anthropic Claude API (חובה — זה "המוח")
- **מודל:** Claude Opus
- **עלות:** ~$0.005–0.01 לשאלה (עם Prompt Caching אפשר לחסוך ~90% על ה-schema)
- **כרגע:** נקנו $5 credits
- ⚠️ **חשוב:** זה **API נפרד** מהמנוי של claude.ai ($20). המנוי לא נותן גישת API.

### 4.2 הרצת ה-Backend בענן

| פלטפורמה | עלות | הערה |
|-----------|------|------|
| **Render** (כרגע) | חינם / $7 לחודש ל-always-on | לא מתאים לסביבת GCP שלכם |
| **Google Cloud Run** (מומלץ) | **כמעט $0** — ראה למטה | החלופה הנכונה ל-GCP |

#### עלות Google Cloud Run בפירוט
Cloud Run מתומחר לפי שימוש בפועל, עם **שכבה חינמית נדיבה כל חודש**:
- 2 מיליון בקשות / חודש — חינם
- 360,000 GB-שניות זיכרון — חינם
- 180,000 vCPU-שניות — חינם
- **Scale-to-zero**: כשאף אחד לא משתמש → 0 קונטיינרים → **0 עלות**

➡️ **למעשה:** צ'אט בנפח של מאות–אלפי שאלות בחודש **נשאר בתוך השכבה החינמית = ~$0/חודש.**
רק בנפחים גבוהים מאוד מתחילים לשלם, וגם אז מדובר בסנטים.

> אם הכוונה הייתה ל-**Cloud Code** (תוסף ה-IDE של גוגל לפיתוח/דיפלוי) — הוא **חינמי** לחלוטין. העלות היחידה היא של המשאבים שמריצים בפועל (Cloud Run למעלה).

### 4.3 עלות ה-MCP של Qlik — הבהרה חשובה ⭐
**ה-MCP של Qlik עולה $0 בפרודקשן — כי הוא בכלל לא חלק מהצ'אט.**

יש כאן בלבול נפוץ, אז נחדד:
- **מה זה ה-MCP:** כלי ש**אני (Claude) השתמשתי בו במהלך הפיתוח** — דרך הסשן הזה — כדי לגלות את ה-schema האמיתי של האפליקציה (השדות והמדדים).
- **האם הצ'אט בפרודקשן משתמש ב-MCP?** **לא.** הצ'אט שבנינו ניגש לנתונים דרך **Qlik Capability API בדפדפן** — לא דרך MCP.
- **לכן:** ה-MCP **לא מוסיף שום עלות שוטפת** למוצר. הוא היה כלי פיתוח חד-פעמי.
- **מצד Qlik:** ה-MCP משתמש ב-Qlik Cloud APIs שכלולים במנוי Qlik הקיים — אין חיוב נפרד.
- **מצד Anthropic:** השימוש ב-MCP הוא חלק מצריכת ה-tokens הרגילה של שיחת Claude — אין תעריף נפרד ל-MCP.

| רכיב | עלות שוטפת בפרודקשן |
|------|----------------------|
| Qlik MCP | **$0** (כלי פיתוח, לא במוצר) |
| Claude API | ~$0.005–0.01 לשאלה |
| Cloud Run | ~$0 (שכבה חינמית) |
| Qlik Cloud | המנוי הקיים |
| GitHub | חינם |

## 6. מעבר לפרודקשן ב-GCP (Cloud Run)

### שלבי ה-deploy
1. הוספת `Dockerfile` לפרויקט (או deploy מ-source ישירות).
2. שמירת מפתח Claude ב-**Secret Manager** (לא בקוד!).
3. `gcloud run deploy` → מקבלים URL בצורת `https://xxx.run.app`.
4. עדכון ה-URL ב-`chatbot.js`.
5. הוספת ה-URL החדש ל-**CSP של Qlik** (כמו שעשינו ל-Render).

### הרשאות IAM שצריך

| למי | תפקיד (IAM Role) | בשביל מה |
|-----|------------------|----------|
| המשתמש שמבצע deploy | `Cloud Run Admin` + `Cloud Build Editor` | להעלות ולפרוס את הקוד |
| ה-Service Account של השירות | `Secret Manager Secret Accessor` | לקרוא את מפתח Claude מ-Secret Manager |
| מי שקורא ל-Backend | `Cloud Run Invoker` (או public) | גישה ל-endpoint |

> ⚠️ ה-Backend **רק קורא ל-Claude API** — הוא **לא** ניגש ל-BigQuery או לנתונים.
> לכן הוא **לא צריך** הרשאות BigQuery. שליפת הנתונים קורית ב-Qlik, בצד הדפדפן,
> עם ההרשאות של המשתמש עצמו.

## 6א. Qlik — הרשאות והגדרות נדרשות (מפורט) ⭐

יש כאן **שני עולמות נפרדים** של הרשאות Qlik: אחד עבור **ה-Extension בפרודקשן**,
ואחד עבור **ה-MCP בפיתוח**. חשוב לא לבלבל ביניהם.

### 6א.1 הגדרות ברמת ה-Tenant (דורש Tenant Admin)
הפעולות הבאות נעשות ב-**Management Console** ודורשות תפקיד **Tenant Admin**:

| הגדרה | היכן | בשביל מה | חובה? |
|--------|------|----------|--------|
| **Content Security Policy** | Governance → Content security policy | להוסיף את דומיין ה-Backend ל-`connect-src` כדי שה-Extension יוכל לקרוא אליו | ✅ חובה ל-Extension |
| **API keys — הפעלה** | Settings → API keys | להפעיל יצירת API keys ב-tenant (אחרת אי אפשר ליצור מפתח ל-MCP) | ✅ חובה ל-MCP |
| **AI features** | Settings → אזור ה-AI / "Machine learning & AI" | להפעיל/לכבות יכולות AI של Qlik (Insight Advisor, Qlik Answers, Assistant) | ⚪ רק אם משתמשים ב-AI של Qlik עצמו |
| **Large Language Model usage** | Settings (תחת AI) | אישור שימוש ב-LLM בתוך Qlik | ⚪ רק ל-AI הפנימי של Qlik |

> 💡 **הבהרה חשובה לגבי "AI Features" של Qlik:**
> הצ'אט שבנינו **לא משתמש** ב-AI features של Qlik — אנחנו משתמשים ב-**Claude API חיצוני**.
> לכן הפעלת "AI features" ב-Qlik **אינה חובה** עבור הצ'אט שלנו.
> צריך אותה **רק** אם בעתיד תרצו להשתמש ב-Qlik Answers / Insight Advisor / Assistant
> המובנים של Qlik (מוצר נפרד, בתשלום נוסף לפי המנוי).

### 6א.2 הרשאות עבור ה-Extension (פרודקשן)

| מי | הרשאה ב-Qlik | בשביל מה |
|----|---------------|----------|
| מי שמעלה את ה-Extension | יכולת **Upload extensions** (משתמש Professional/Full עם הרשאת פיתוח, או Admin) | להעלות את `chatbot.zip` |
| מי שעורך את ה-Sheet ומוסיף את ה-Extension | **Can edit** במרחב (Space) של האפליקציה | להוסיף את הצ'אט ל-sheet |
| המשתמש הסופי (צופה) | **Can view** באפליקציה + במרחב | להשתמש בצ'אט ולראות נתונים |

> ה-Extension רץ עם ה-**session של המשתמש הצופה**. כלומר אין צורך בהרשאות מיוחדות
> ל-Extension עצמו — הוא יורש את הרשאות הצפייה של המשתמש על האפליקציה והנתונים.

### 6א.3 הרשאות עבור ה-MCP של Qlik (פיתוח בלבד)

ה-MCP הוא הכלי שבו השתמשתי לגלות את ה-schema. כדי שיעבוד צריך:

| מה | הרשאה / פעולה | בשביל מה |
|----|----------------|----------|
| הפעלת API keys ב-tenant | Tenant Admin מפעיל ב-Management Console | תנאי מקדים ליצירת מפתח |
| יצירת API key אישי | יכולת **Generate API keys** למשתמש | ה-MCP מתחבר ל-Qlik עם המפתח הזה |
| גישת המפתח/הזהות לאפליקציות | **Can view / Can consume data** במרחבים הרלוונטיים | שה-MCP יוכל לקרוא apps, fields, measures |
| (חלופה ל-API key) **OAuth client** | Admin יוצר OAuth M2M client ב-Settings → OAuth | חיבור מאובטח יותר למכונה |

> ⚠️ **תזכורת:** ה-MCP הוא **כלי פיתוח**, לא חלק מהמוצר. בפרודקשן הצ'אט **לא** משתמש בו.
> ההרשאות בסעיף הזה נדרשות רק אם תרצו להמשיך להשתמש ב-MCP לפיתוח/גילוי schema.

### 6א.4 סיכום: מה חובה לצ'אט שלנו לעומת מה אופציונלי

| הגדרת Qlik | נדרש לצ'אט שלנו? |
|-------------|------------------|
| CSP whitelist ל-Backend | ✅ **חובה** |
| Upload extension + הרשאות Space | ✅ **חובה** |
| API keys (ל-MCP) | ⚪ רק לפיתוח, לא לפרודקשן |
| AI features / LLM של Qlik | ❌ **לא נדרש** (אנחנו על Claude חיצוני) |

## 7. אבטחה — לפני שהופכים את ה-Sheet ל-PUBLIC ⚠️

**אם הופכים את ה-Sheet ל-public כרגע — כולם יוכלו להשתמש בצ'אט, וזו בעיה.**

### מה חשוף ומה לא

| פריט | בטוח? | הערה |
|------|--------|------|
| מפתח Claude | ✅ בטוח | יושב בצד שרת, לא נחשף בדפדפן |
| כתובת ה-Backend | ❌ גלויה | מופיעה בקוד ה-JS של ה-Extension |
| ה-Backend עצמו | ❌ פתוח | אין אימות, CORS פתוח ל-"*" |

### הסיכון
גורם זדוני יכול לקחת את כתובת ה-backend ולשלוח אלפי בקשות → **לשרוף את ה-Claude credits**.

### מה צריך להוסיף לפני public
1. **אימות** בין Extension ל-Backend (טוקן משותף / מפתח).
2. **Rate limiting** — הגבלת בקשות לכל משתמש/IP.
3. **CORS מוגבל** — רק לדומיין של Qlik (לא "*").
4. **תקרת עלות** ב-Anthropic — הגבלת תקציב חודשי מקסימלי.

### לגבי הנתונים
ה-Extension משתמש ב-**session של המשתמש ב-Qlik** (`qlik.openApp`). כלומר משתמש רואה רק
נתונים שיש לו הרשאה אליהם. אם ה-Sheet anonymous/public — חלות הרשאות המשתמש האנונימי;
צריך לוודא מה בדיוק הוא רואה.

## 8. צעדים הבאים מומלצים
1. **העברה ל-Cloud Run** + מפתח ב-Secret Manager (פתרון GCP אמיתי).
2. **הוספת שכבת אבטחה** ל-Backend (סעיף 7) — חובה לפני public.
3. **הרחבת המדדים** — הזנת 40+ ה-measures המוכנים ל-Claude לכיסוי רחב יותר.
4. **תמיכה ריבוי-אפליקציות** — semantic router שבוחר את ה-app הנכון לפי השאלה (יש לכם עשרות apps).
```
